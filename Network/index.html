네트워크

네트워크 소개 네트워크 모델 물리 계층 데이터 링크 계층 네트워크 계층 전송 계층 세션 계층 표현 계층 응용 계층 주요 프로토콜 네트워크 보안 무선 네트워크 네트워크 문제해결

네트워크 소개

네트워크는 컴퓨터와 다른 장치들이 통신 매체를 통해 서로 연결되어 데이터를 주고받을 수 있는 시스템입니다. 현대 사회에서 네트워크는 우리의 일상생활과 비즈니스 활동에 필수적인 요소가 되었습니다.

네트워크의 주요 목적

  • 정보와 자원의 공유
  • 통신 및 데이터 교환
  • 원격 접속 및 제어
  • 분산 처리 지원
  • 비용 절감 및 효율성 향상

네트워크의 종류

규모에 따른 분류

  • LAN (Local Area Network): 가정, 사무실, 학교 등 제한된 지역 내의 네트워크
  • MAN (Metropolitan Area Network): 도시 규모의 네트워크
  • WAN (Wide Area Network): 국가, 대륙, 전 세계를 연결하는 네트워크

토폴로지에 따른 분류

  • 버스형(Bus): 모든 노드가 하나의 통신 라인에 연결
  • 스타형(Star): 중앙 허브를 중심으로 연결
  • 링형(Ring): 각 노드가 양옆의 두 노드와 연결되어 원형 구조 형성
  • 메시형(Mesh): 모든 노드가 서로 직접 연결

네트워크 모델

네트워크 모델은 네트워크 통신의 복잡성을 관리하기 위해 계층화된 구조를 제공합니다. 가장 널리 알려진 두 가지 네트워크 모델은 OSI 7계층 모델과 TCP/IP 4계층 모델입니다.

OSI 7계층 모델

OSI(Open Systems Interconnection) 모델은 국제표준화기구(ISO)에서 제정한 네트워크 통신의 표준 모델로, 7개의 계층으로 구성됩니다.

계층 이름 주요 기능 주요 장비/프로토콜
7 응용 계층
(Application) 		사용자와 네트워크 간의 인터페이스 제공 HTTP, SMTP, FTP, DNS
6 표현 계층
(Presentation) 		데이터 형식 변환, 암호화, 압축 JPEG, MPEG, SSL/TLS
5 세션 계층
(Session) 		통신 세션 설정, 유지, 종료 NetBIOS, RPC
4 전송 계층
(Transport) 		신뢰성 있는 데이터 전송, 흐름 제어 TCP, UDP
3 네트워크 계층
(Network) 		경로 설정, 논리적 주소 지정 IP, ICMP, 라우터
2 데이터 링크 계층
(Data Link) 		물리적 주소 지정, 오류 검출 이더넷, MAC, 스위치
1 물리 계층
(Physical) 		비트 전송, 케이블, 커넥터 등 물리적 매체 허브, 리피터, 케이블

TCP/IP 4계층 모델

TCP/IP 모델은 인터넷의 기본이 되는 모델로, OSI 모델보다 단순화된 4개의 계층으로 구성됩니다.

응용 계층 (Application Layer)

OSI 모델의 5~7계층(응용, 표현, 세션)에 해당

주요 프로토콜: HTTP, FTP, SMTP, DNS

전송 계층 (Transport Layer)

OSI 모델의 4계층(전송)에 해당

주요 프로토콜: TCP, UDP

인터넷 계층 (Internet Layer)

OSI 모델의 3계층(네트워크)에 해당

주요 프로토콜: IP, ICMP, ARP

네트워크 접근 계층 (Network Access Layer)

OSI 모델의 1~2계층(물리, 데이터 링크)에 해당

주요 프로토콜: 이더넷, Wi-Fi

물리 계층

물리 계층은 OSI 모델의 1계층으로, 실제 데이터가 네트워크를 통해 전송될 수 있도록 물리적 매체를 통한 비트 전송을 담당합니다.

주요 기능

  • 비트스트림을 전기적, 광학적 또는 무선 신호로 변환
  • 통신 매체와 장치 간의 물리적 연결 정의
  • 데이터 전송률, 신호 전압 레벨 등 정의
  • 전송 매체의 활성화 및 비활성화

전송 매체

유선 매체

  • 동축 케이블: 중앙 구리선을 절연체와 외부 금속 쉴드로 감싼 케이블
  • TP(Twisted Pair) 케이블: 두 구리선을 꼬아서 만든 케이블 (UTP, STP)
  • 광섬유 케이블: 빛을 이용해 데이터를 전송하는 케이블

무선 매체

  • 라디오파(RF): Wi-Fi, 블루투스 등
  • 마이크로파: 위성 통신, 마이크로파 중계기
  • 적외선(IR): 짧은 거리 통신에 사용

주요 장비

  • 리피터(Repeater): 신호 감쇠를 해결하기 위해 신호를 재생성하여 전송
  • 허브(Hub): 여러 장치를 연결하고 수신한 데이터를 모든 포트로 전송
  • 네트워크 인터페이스 카드(NIC): 컴퓨터와 네트워크를 연결하는 하드웨어

데이터 링크 계층

데이터 링크 계층은 OSI 모델의 2계층으로, 물리적 연결을 통해 인접한 네트워크 노드 간의 데이터 전송을 담당합니다.

주요 기능

  • 물리적 주소 지정 (MAC 주소)
  • 프레임화 (Framing) - 데이터를 프레임 단위로 분할
  • 오류 감지 및 제어
  • 데이터 흐름 제어
  • 물리 매체에 대한 접근 제어 (MAC 프로토콜)

MAC 주소

MAC(Media Access Control) 주소는 네트워크 인터페이스의 고유 식별자로, 48비트(6바이트) 길이의 하드웨어 주소입니다.

  • 16진수로 표현 (예: 00:1A:2B:3C:4D:5E)
  • 앞 24비트(3바이트)는 제조업체 식별 코드(OUI)
  • 뒤 24비트(3바이트)는 제조업체가 할당한 고유 번호
  • 물리적으로 변경 불가능한 값 (일부 장치에서는 소프트웨어적으로 변경 가능)

이더넷 (Ethernet)

이더넷은 가장 널리 사용되는 LAN 기술로, IEEE 802.3 표준을 따릅니다.

이더넷 프레임 구조

  • Preamble: 프레임의 시작을 알리는 비트 패턴 (8바이트)
  • 목적지 MAC 주소: 수신측 장치의 MAC 주소 (6바이트)
  • 출발지 MAC 주소: 송신측 장치의 MAC 주소 (6바이트)
  • 유형/길이: 상위 계층 프로토콜 유형 또는 데이터 길이 (2바이트)
  • 데이터: 전송할 실제 데이터 (46~1500바이트)
  • FCS: 프레임 오류 검사를 위한 체크섬 (4바이트)

주요 장비

  • 스위치(Switch): MAC 주소를 기반으로 프레임을 특정 포트로 전달하는 장비
  • 브리지(Bridge): 두 네트워크 세그먼트를 연결하고, 필요한 경우에만 프레임을 전달

네트워크 계층

네트워크 계층은 OSI 모델의 3계층으로, 서로 다른 네트워크 간의 데이터 라우팅과 패킷 전달을 담당합니다.

주요 기능

  • 논리적 주소 지정 (IP 주소)
  • 라우팅 - 최적 경로 결정
  • 패킷 포워딩
  • 네트워크 간 연결
  • 혼잡 제어

IP (Internet Protocol)

IP는 인터넷의 핵심 프로토콜로, 데이터 패킷을 출발지에서 목적지까지 전달하는 비연결형 프로토콜입니다.

IP 주소

IPv4
  • 32비트 주소 체계
  • 점으로 구분된 10진수 표기법 (예: 192.168.0.1)
  • 클래스 기반 (A, B, C, D, E) 또는 CIDR 방식으로 구분
  • 약 43억 개의 주소 공간 (이론적)
IPv6
  • 128비트 주소 체계
  • 콜론으로 구분된 16진수 표기법 (예: 2001:0db8:85a3:0000:0000:8a2e:0370:7334)
  • IPv4 고갈 문제 해결을 위해 도입
  • 보안 기능 내장, 자동 구성 기능 강화

서브넷팅 (Subnetting)

대규모 네트워크를 작은 네트워크로 분할하는 기법으로, 네트워크 관리와 라우팅 효율성을 높일 수 있습니다.

  • 서브넷 마스크를 사용하여 네트워크 부분과 호스트 부분 구분
  • 예시: 192.168.1.0/24는 192.168.1.0 네트워크의 24비트가 네트워크 부분임을 의미
  • CIDR(Classless Inter-Domain Routing) 표기법 사용

라우팅 (Routing)

라우팅은 패킷이 출발지에서 목적지까지 가는 최적의 경로를 결정하는 과정입니다.

라우팅 프로토콜 종류

거리 벡터 라우팅 프로토콜
  • RIP (Routing Information Protocol)
  • 홉 수를 기준으로 경로 결정
  • 주기적으로 전체 라우팅 테이블 교환
링크 상태 라우팅 프로토콜
  • OSPF (Open Shortest Path First)
  • Dijkstra 알고리즘 사용
  • 변경된 부분만 업데이트
  • 대규모 네트워크에 적합

주요 장비

  • 라우터(Router): 서로 다른 네트워크를 연결하고 패킷을 전달하는 장비
  • L3 스위치: 하드웨어 기반의 고속 라우팅 기능을 가진 스위치

전송 계층

전송 계층은 OSI 모델의 4계층으로, 종단 간(end-to-end) 통신을 관리하고 데이터의 신뢰성 있는 전달을 보장합니다.

주요 기능

  • 세그먼트화 - 데이터를 세그먼트 단위로 분할
  • 흐름 제어 - 수신자의 처리 속도에 맞게 데이터 전송
  • 오류 검출 및 복구
  • 연결 지향/비연결 지향 서비스 제공
  • 혼잡 제어 - 네트워크 과부하 방지
  • 포트 번호를 통한 프로세스 구분

TCP (Transmission Control Protocol)

신뢰성 있는 연결 지향 프로토콜로, 데이터의 순서와 전달을 보장합니다.

  • 연결 설정을 위한 3-way 핸드셰이크
  • 데이터 전송 보장 (재전송 메커니즘)
  • 흐름 제어 및 혼잡 제어
  • 순서 보장 (시퀀스 번호 사용)
  • 오류 검출 및 복구
  • HTTP, FTP, SMTP 등에서 사용

UDP (User Datagram Protocol)

단순하고 빠른 비연결 지향 프로토콜로, 신뢰성보다 속도가 중요한 경우 사용됩니다.

  • 연결 설정 없음 (핸드셰이크 없음)
  • 데이터 전송 보장 없음 (재전송 없음)
  • 순서 보장 없음
  • 헤더 오버헤드가 적음 (8바이트)
  • 간단한 오류 검출 (체크섬)
  • DNS, DHCP, 스트리밍, 게임 등에서 사용

포트 (Port)

포트는 특정 호스트 내의 프로세스를 식별하는 16비트 숫자로, 동일한 IP 주소에서 여러 서비스를 구분하는 데 사용됩니다.

주요 포트 번호

  • 20, 21: FTP (파일 전송)
  • 22: SSH (보안 쉘)
  • 23: Telnet
  • 25: SMTP (이메일 전송)
  • 53: DNS (도메인 이름 해석)
  • 80: HTTP (웹)
  • 443: HTTPS (보안 웹)
  • 110: POP3 (이메일 수신)
  • 143: IMAP (이메일 수신)
  • 3306: MySQL

세션 계층

세션 계층은 OSI 모델의 5계층으로, 통신 세션의 설정, 유지, 종료를 관리하고 동기화를 담당합니다.

주요 기능

  • 세션 설정, 유지, 종료
  • 세션 복구 (체크포인트, 동기화)
  • 통신 모드 관리 (전이중, 반이중, 단방향)
  • 대화 제어 (대화 시작, 데이터 전송, 대화 종료)

주요 프로토콜 및 서비스

NetBIOS (Network Basic Input/Output System)

  • 로컬 네트워크에서 컴퓨터 간 통신을 위한 API
  • 이름 서비스, 세션 서비스, 데이터그램 서비스 제공
  • Windows 네트워킹에서 주로 사용

RPC (Remote Procedure Call)

  • 원격 시스템에서 프로시저 호출을 가능하게 하는 프로토콜
  • 분산 컴퓨팅 환경에서 사용
  • 클라이언트-서버 모델 기반

세션과 쿠키

웹 애플리케이션에서 세션 관리는 주로 세션 ID와 쿠키를 통해 이루어집니다.

  • 세션(Session): 서버에 저장되는 사용자 정보
  • 쿠키(Cookie): 클라이언트에 저장되는 작은 데이터 조각
  • 세션 ID는 쿠키를 통해 클라이언트에 전달되고, 이후 요청에서 포함됨
  • 로그인 상태 유지, 장바구니 정보 저장 등에 활용

표현 계층

표현 계층은 OSI 모델의 6계층으로, 데이터의 형식, 암호화, 압축 등 응용 계층과 통신하기 위한 데이터 변환을 담당합니다.

주요 기능

  • 데이터 형식 변환 (문자 인코딩, 데이터 표현 형식)
  • 암호화 및 복호화
  • 데이터 압축 및 압축 해제
  • 데이터 표준화

주요 기술 및 표준

문자 인코딩

  • ASCII: 7비트 영문자 인코딩
  • Unicode: 전 세계 문자를 위한 국제 표준
  • UTF-8: 가변 길이 문자 인코딩 (1~4바이트)
  • UTF-16: 2바이트 또는 4바이트 문자 인코딩

데이터 형식

  • XML: 구조화된 데이터 표현 형식
  • JSON: 경량 데이터 교환 형식
  • YAML: 사람이 읽기 쉬운 데이터 직렬화 형식
  • Protobuf: 구조화된 데이터의 효율적인 직렬화

암호화

  • SSL/TLS: 인터넷 통신 보안을 위한 암호화 프로토콜
  • 대칭키 암호화: AES, DES, 3DES
  • 비대칭키 암호화: RSA, ECC
  • 해시 함수: SHA, MD5

데이터 압축

  • 무손실 압축: ZIP, GZIP, LZ77
  • 손실 압축: JPEG, MP3, MP4
  • HTTP 압축: 웹 페이지 전송 시 사용

응용 계층

응용 계층은 OSI 모델의 7계층으로, 사용자와 직접 상호작용하는 네트워크 애플리케이션에 서비스를 제공합니다.

주요 기능

  • 사용자 인터페이스 제공
  • 애플리케이션 간 통신 지원
  • 파일 전송, 이메일, 웹 브라우징 등 다양한 서비스 제공
  • 네트워크 자원에 대한 접근 관리

주요 프로토콜 및 서비스

웹 관련

  • HTTP/HTTPS: 웹 페이지 전송
  • WebSocket: 양방향 통신
  • HTTP/2, HTTP/3: 개선된 웹 프로토콜

이메일 관련

  • SMTP: 이메일 전송
  • POP3: 이메일 수신 (다운로드)
  • IMAP: 이메일 수신 (서버 보관)

파일 공유

  • FTP: 파일 전송
  • SFTP: 보안 파일 전송
  • SMB/CIFS: 파일 공유

도메인 이름 시스템 (DNS)

DNS는 도메인 이름(예: www.example.com)을 IP 주소(예: 93.184.216.34)로 변환하는 시스템입니다.

DNS 구조

  • 계층적 구조: 루트, 최상위 도메인(TLD), 2차 도메인, 서브도메인
  • 분산 데이터베이스: 전 세계 DNS 서버에 분산 저장
  • 캐싱: 반복적인 조회 요청 감소
  • 레코드 타입: A(IPv4), AAAA(IPv6), CNAME(별칭), MX(메일), NS(네임서버) 등

주요 프로토콜

네트워크 프로토콜은 장치 간 데이터 교환을 위한 규칙과 표준을 정의합니다. 여기서는 여러 계층에서 사용되는 주요 프로토콜에 대해 알아봅니다.

HTTP (Hypertext Transfer Protocol)

웹 페이지와 데이터를 전송하기 위한 응용 계층 프로토콜입니다.

HTTP 메소드

  • GET: 리소스 조회
  • POST: 리소스 생성
  • PUT: 리소스 수정
  • DELETE: 리소스 삭제
  • HEAD: 헤더 정보만 요청
  • OPTIONS: 지원 메소드 확인

HTTP 상태 코드

  • 1xx: 정보성 응답
  • 2xx: 성공 (200 OK, 201 Created)
  • 3xx: 리다이렉션 (301 Moved Permanently, 302 Found)
  • 4xx: 클라이언트 오류 (404 Not Found, 403 Forbidden)
  • 5xx: 서버 오류 (500 Internal Server Error, 503 Service Unavailable)

HTTPS (HTTP Secure)

HTTP의 보안 버전으로, TLS/SSL을 사용하여 데이터를 암호화합니다.

  • 데이터 암호화로 도청 및 중간자 공격 방지
  • 인증서를 통한 서버 신원 확인
  • 데이터 무결성 보장
  • 검색 엔진 최적화(SEO) 이점

DHCP (Dynamic Host Configuration Protocol)

네트워크 장치에 IP 주소를 자동으로 할당하는 프로토콜입니다.

DHCP 작동 과정

  1. DHCP Discover: 클라이언트가 DHCP 서버 탐색
  2. DHCP Offer: 서버가 사용 가능한 IP 제안
  3. DHCP Request: 클라이언트가 제안된 IP 요청
  4. DHCP Acknowledge: 서버가 IP 사용 확인

ARP (Address Resolution Protocol)

IP 주소를 MAC 주소로 변환하는 프로토콜입니다.

  • 로컬 네트워크에서 통신할 때 필요
  • ARP 캐시에 매핑 정보 저장
  • 브로드캐스트를 통해 MAC 주소 요청

ICMP (Internet Control Message Protocol)

네트워크 상태 정보와 오류 메시지를 전달하는 프로토콜입니다.

  • Ping(Echo Request/Reply)에 사용
  • Traceroute에 사용 (TTL 초과 메시지)
  • 목적지 도달 불가, 포트 도달 불가 등 오류 보고

네트워크 보안

네트워크 보안은 네트워크 인프라와 데이터를 보호하기 위한 정책, 조치, 기술의 조합입니다.

주요 위협

악성 공격

  • DoS/DDoS: 서비스 거부 공격
  • 중간자 공격: 통신 가로채기
  • 포트 스캐닝: 취약점 탐색
  • 패킷 스니핑: 데이터 도청

악성 소프트웨어

  • 바이러스: 실행 파일에 감염
  • 웜: 자가 복제 및 확산
  • 트로이 목마: 정상 프로그램으로 위장
  • 랜섬웨어: 데이터 암호화 후 금전 요구

보안 기술

방화벽 (Firewall)

  • 네트워크 트래픽 필터링
  • 패킷 규칙 기반 통제
  • 네트워크 경계 보호

VPN (Virtual Private Network)

  • 암호화된 터널 통신
  • 원격 접속 보안
  • 프라이버시 보호

IDS/IPS

  • 침입 탐지 및 방지
  • 이상 트래픽 감지
  • 실시간 대응

암호화 기술

  • 대칭키 암호화: 동일한 키로 암호화와 복호화
    • AES, DES, 3DES
    • 빠르지만 키 공유 문제 존재
  • 비대칭키 암호화: 공개키/개인키 쌍 사용
    • RSA, ECC, Diffie-Hellman
    • 키 교환 문제 해결, 디지털 서명에 사용
  • 해시 함수: 단방향 데이터 변환
    • SHA-256, SHA-3, MD5(약함)
    • 데이터 무결성 검증, 패스워드 저장

무선 네트워크

무선 네트워크는 유선 케이블 대신 전파나 적외선 등의 무선 신호를 사용하여 데이터를 전송하는 네트워크입니다.

무선 LAN (WLAN)

무선 랜은 IEEE 802.11 표준을 사용하는 로컬 영역 무선 네트워크로, 흔히 Wi-Fi라고 불립니다.

표준 최대 속도 주파수 대역 특징
802.11b 11 Mbps 2.4 GHz 간섭에 취약
802.11a 54 Mbps 5 GHz 더 적은 간섭, 짧은 범위
802.11g 54 Mbps 2.4 GHz 802.11b와 호환
802.11n 600 Mbps 2.4/5 GHz MIMO 기술, 이중 대역
802.11ac ~3.5 Gbps 5 GHz MU-MIMO, 넓은 채널 대역폭
802.11ax (Wi-Fi 6) ~9.6 Gbps 2.4/5/6 GHz OFDMA, 향상된 효율성

무선 네트워크 보안

무선 보안 프로토콜

  • WEP: 오래되고 취약한 암호화 (사용 지양)
  • WPA: WEP 개선, TKIP 사용
  • WPA2: AES 암호화 사용, 현재 표준
  • WPA3: 향상된 암호화, SAE 인증

보안 관행

  • 기본 SSID 및 관리자 비밀번호 변경
  • 강력한 암호 사용 (WPA2/WPA3)
  • 게스트 네트워크 분리
  • 펌웨어 정기 업데이트
  • MAC 주소 필터링 (보조 보안 방식)

기타 무선 기술

블루투스 (Bluetooth)

  • 단거리 무선 통신 기술
  • 2.4 GHz ISM 대역 사용
  • 개인 영역 네트워크(PAN) 구성
  • 최신 버전: Bluetooth 5.3

NFC (Near Field Communication)

  • 극단거리 무선 통신 (10cm 이내)
  • 13.56 MHz 주파수 대역
  • 모바일 결제, 접근 제어
  • 수동형 및 능동형 장치 지원

5G

  • 5세대 셀룰러 이동통신
  • 초고속 (최대 20Gbps)
  • 초저지연 (1ms 목표)
  • 대규모 IoT 연결 지원

네트워크 문제해결

네트워크 문제해결은 네트워크 연결 및 성능 문제를 식별하고 해결하는 과정입니다.

문제해결 도구

명령줄 도구

  • ping: ICMP 에코 요청으로 연결 확인
  • traceroute/tracert: 패킷 경로 추적
  • ipconfig/ifconfig: IP 구성 정보 확인
  • nslookup/dig: DNS 조회 테스트
  • netstat: 네트워크 연결 상태 확인

소프트웨어 도구

  • Wireshark: 패킷 분석 및 캡처
  • 네트워크 모니터링 도구: Nagios, PRTG
  • 침투 테스팅 도구: Nmap, Metasploit
  • 로그 분석 도구: ELK Stack, Splunk

일반적인 문제 및 해결 방법

연결 문제

  • 인터넷 연결 불가
    • 물리적 연결 확인 (케이블, 전원 등)
    • IP 설정 확인 (DHCP, 고정 IP)
    • DNS 서버 확인
    • 라우터/모뎀 재시작
  • 간헐적 연결 끊김
    • 간섭 확인 (무선의 경우)
    • 네트워크 부하 확인
    • 하드웨어 문제 확인
  • 느린 네트워크
    • 대역폭 사용량 모니터링
    • 네트워크 병목 현상 확인
    • 하드웨어 성능 확인
    • 악성 프로그램 검사

문제해결 방법론

  1. 문제 정의

    정확히 어떤 문제가 발생했는지 명확히 정의

  2. 정보 수집

    로그, 오류 메시지, 사용자 보고 등 관련 정보 수집

  3. 계층별 접근

    물리 계층부터 시작하여 상위 계층으로 진행 (OSI 모델 활용)

  4. 변수 제어

    한 번에 하나의 변수만 변경하며 테스트

  5. 해결책 구현 및 검증

    해결책 적용 후 문제가 해결되었는지 확인

  6. 문서화

    문제와 해결 과정을 기록하여 향후 참조
